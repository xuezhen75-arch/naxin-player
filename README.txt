# 纳心音乐播放器 v1.0 — 使用说明

## 文件结构

```
naxin-player/
├── index.html          ← 主播放器（双击浏览器打开）
└── README.txt          ← 本文件
```

> 无需单独的 audio 文件夹！所有音频文件通过后台上传到浏览器 IndexedDB 存储。

## 快速开始

1. 双击 `index.html` 在浏览器中打开
2. 首次使用：点击右上角「⚙ 管理」→ 输入密码 7856 → 登录
3. 上传环境音场景（MP3/OGG/WAV/AAC/M4A/FLAC/WebM）
4. 上传音乐曲目分轨（同上格式，可多选8个文件）
5. 关闭管理面板，回到主界面选择场景和曲目
6. 点击底部 ▶ 播放

## 后台管理（密码 7856）

| 功能 | 说明 |
|------|------|
| 上传环境音场景 | 输入名称 + 选择音频文件，支持多格式 |
| 管理环境音列表 | 查看/删除已上传场景 |
| 上传音乐曲目 | 输入曲目名 + 多选分轨文件（最多8个） |
| 管理曲目列表 | 查看/删除已上传曲目 |
| 清空数据 | 一键清除所有数据 |
| 导出配置 | 导出场景和曲目列表JSON |

## 支持的音频格式

MP3 / OGG / WAV / AAC / M4A / FLAC / WebM

## 环境音均衡调节（10段）

| 频段 | 频率 | 
|------|------|
| 1 | 32Hz   |
| 2 | 64Hz   |
| 3 | 125Hz  |
| 4 | 250Hz  |
| 5 | 500Hz  |
| 6 | 1kHz   |
| 7 | 2kHz   |
| 8 | 4kHz   |
| 9 | 8kHz   |
| 10| 16kHz  |

完全对齐 myNoise 标准，用 BiquadFilter 带通滤波器实时切分环境音录音。

## 音乐分轨（8路）

- 纯数字标识：第1路 / 第2路 / ... / 第8路
- 上传时文件顺序随机分配（用户凭耳朵调节，不标注乐器名）
- 每路独立 GainNode 控制音量

## 独立部署

- 纯静态 HTML，可部署到任何静态服务器
- 数据存储在浏览器 IndexedDB（无需后端数据库）
- 可部署到 Render / EdgeOne Pages / GitHub Pages / 任意CDN

## 预留：音谋加H5对接接口

代码中已预留以下接口（在 script 末尾注释中）：
- POST /api/yinmoujia/sync — 同步用户登录态、曲目库、预设
- GET /api/yinmoujia/tracks — 从音谋加云端拉取曲目列表
- window.naxinAPI — 全局API对象

对接时解除注释即可使用。

## 技术参数

| 项目 | 规格 |
|------|------|
| 环境音均衡 | 10段 BiquadFilter，32Hz~16kHz |
| 音乐分轨 | 8路独立 AudioBufferSourceNode + GainNode |
| 动画 LFO | 3种模式 Soft/Hard/Solo，速度 0.125x~8x |
| 存储 | IndexedDB（本地浏览器数据库） |
| 主题 | 3色 CSS变量切换 |
| 布局 | 手机优先，上下分区 |
| 技术栈 | 纯前端 HTML+CSS+JS，Web Audio API，零依赖 |
